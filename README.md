# store-zip

圧縮ファイル（ZIP / RAR / 7z など）を **無圧縮アーカイブ** に変換する Windows 用 SendTo ツールです。
展開後の即時アクセスや再圧縮を不要にすることを目的として設計されています。

---

## 特徴

| 機能 | 説明 |
|------|------|
| 主要形式に対応 | ZIP / RAR / RAR5 / 7z / tar.gz など |
| 出力形式を選択可能 | 無圧縮 ZIP（デフォルト）または 無圧縮 RAR + リカバリーレコード |
| ゴミファイル自動削除 | `Thumbs.db` / `.DS_Store` / `.url` など設定で管理 |
| 未分類ファイルの対話処理 | 初回のみ確認→拡張子単位で `decisions.json` に自動保存 |
| 文字コード対応 | Shift-JIS / UTF-8 / その他を自動判定。旧日本語 ZIP も正常処理 |
| ルートフォルダ除去 | 単一ルートフォルダのみの場合、フォルダを剥がしてフラット化 |
| 元ファイル保護 | 変換成功後に元ファイルをゴミ箱へ送る。失敗時も完全削除へはフォールバックしない |
| 一括・並列処理 | 複数ファイルを同時に選択して送信可能（並列数は `1〜4` で設定可能。シリーズ順番処理は `off / strict / fuzzy` で切替可能） |
| NAS / UNC パス対応 | ネットワークドライブ上のファイルも直接処理可能 |
| スリープ抑止 | 処理中は Windows のスリープを防止（NAS 接続切断対策） |
| 結果サマリー | 処理完了後に変換 / スキップ / エラーの件数を一覧表示 |

## インストール

### 1. Python と 7-Zip をインストール

上記リンクから取得し、それぞれインストールしてください。
7-Zip は標準の場所（`C:\Program Files\7-Zip\`）へのインストールを推奨します。

### 2. `install.bat` を実行

`install.bat` をダブルクリックすると、以下が自動で行われます。

```
[1/4] Python の確認
[2/4] 必要パッケージのインストール（pyinstaller / send2trash / tomli）
[3/4] store-zip.exe のビルド（dist\ フォルダに生成）
[4/4] 右クリックメニュー「送る」への登録
```

> 完了メッセージが出たらインストール成功です。

### 3. 使い方

エクスプローラーで圧縮ファイルを選択（複数可）
→ 右クリック → **「送る」** → **「store-zip」**

変換後のファイルは元ファイルと同じ場所に保存されます。
元ファイルは**ゴミ箱**に移動します（完全には削除されません）。
ゴミ箱への移動に失敗した場合も、完全削除には切り替えません。

### 4. 設定変更

`store-zip.exe` を**直接ダブルクリック**すると設定メニューが表示されます。

```
  [ 1] 出力フォーマット                   zip
  [ 2] タイムスタンプ保持                 有効
  ...
  [s] 保存して終了  [q] 保存せず終了
```

番号を入力して設定を変更し、`s` で保存します。
`config.toml` を直接編集することもできます。

## ファイル構成

```
store-zip/
├── store-zip.py              メインスクリプト
├── config.toml            設定ファイル（ソース）
├── install.bat            インストール用バッチ
├── LICENSE
└── dist/
    ├── store-zip.exe         実行ファイル（ビルド後に生成）
    ├── config.toml        実行時に参照する設定ファイル ← ここを編集する
    ├── decisions.json     ユーザーの判断履歴（自動生成）
    └── store-zip.log         実行ログ（write_log=true のとき生成）
```

> **設定を変更する場合は `dist\config.toml` を編集してください。**
> ルート直下の `config.toml` はソース管理用であり、実行中の exe には反映されません。

## 設定ファイル（config.toml）

`dist\config.toml` をテキストエディタで編集します。変更はすぐに反映されます（再ビルド不要）。

### 主な設定項目

```toml
# 未分類ファイルの処理方針
#   "ask"  : ユーザーに確認する（デフォルト）
#   "keep" : すべて自動保持
#   "junk" : すべて自動削除
unknown_file_action = "ask"

# 同名ファイルが存在する場合の命名規則
#   "counter" : archive (1).zip, archive (2).zip ...
#   "date"    : archive_20241225.zip
duplicate_name_style = "counter"

# 元ファイルのタイムスタンプを引き継ぐか
#   true  : 元ファイルの更新日時を維持する（デフォルト）
#   false : 変換処理を行った日時をそのまま使う
preserve_timestamp = true

# ZIP 内の空フォルダを削除するか
#   true  : 空フォルダを削除する（デフォルト）
#   false : 空フォルダをそのまま維持する
remove_empty_dirs = true

# ログファイルを生成するか（dist\store-zip.log に追記）
write_log = false

# 既に無圧縮ZIPのファイルも、条件に関係なく再処理するか
#   true  : スキップせず毎回再展開・再パックする
#   false : 通常どおり必要時のみ処理する（デフォルト）
force_reprocess_store_zip = false

# シリーズ順番処理の強さ
#   "off"    : まとめ処理しない（デフォルト）
#   "strict" : 明確に同じシリーズ名のものだけ順番処理する
#   "fuzzy"  : 数字以外の名前がよく似たものも順番処理する
series_grouping_mode = "off"

# 並列処理数
#   1 : 常に1件ずつ順番に処理
#   2〜4 : 指定数まで同時処理
parallel_workers = 4

# 処理開始前のファイル一覧の表示件数上限
#   0 : 制限なし（デフォルト）
#   n : 先頭 n 件のみ表示し「... 他 n ファイル」と表示
file_list_limit = 0

# 出力フォーマット
#   "zip" : 無圧縮 ZIP（デフォルト）
#   "rar" : 無圧縮 RAR + リカバリーレコード（WinRAR の rar.exe が必要）
output_format = "zip"

# リカバリーレコードの割合（%）。output_format = "rar" のときのみ有効。
rar_recovery_record = 5

# rar.exe のパスを明示指定する場合のみ設定（空文字 = 自動検索）
# 自動検索順: exe と同じフォルダ → Program Files\WinRAR → PATH
# 例: rar_exe_path = 'D:\Tools\WinRAR\rar.exe'
rar_exe_path = ""

# 同名・同内容の出力ファイルが既に存在する場合の動作
#   true  : 重複とみなし新ファイルを破棄してスキップ（デフォルト）
#           元ファイルの削除失敗による二重変換を防ぐ
#   false : duplicate_name_style に従いリネームして保存
skip_if_same_size = true
```

### 保持・削除パターンの管理

```toml
# 保持を強制するパターン（junk_patterns より優先）
# 画像ファイルはデフォルトで登録済み
allow_patterns = [
    "*.jpg", "*.jpeg", "*.png", "*.gif", ...
]

# ゴミとして削除するファイル名パターン（glob 形式）
junk_patterns = [
    "Thumbs.db", "desktop.ini", ".DS_Store",
    "*.url", "*.lnk", "*.nfo", "*.torrent", ...
]

# ゴミとして削除するフォルダ名パターン
junk_dirs = [
    "__MACOSX", ".git", ".svn",
]
```

## 未分類ファイルの処理（decisions.json）

`allow_patterns` にも `junk_patterns` にも一致しないファイルが見つかると、
**拡張子単位**で確認プロンプトが表示されます。

```
  未分類ファイル: *.txt  （例: readme.txt）
  このファイルは既存のルールに一致しませんでした。
  選択した内容は decisions.json に保存され、次回から自動適用されます。
  [K] 保持する  [D] 削除する
  選択 [K/D]:
```

- 一度決定すると `decisions.json` に保存され、次回以降は確認なしで自動適用されます。
- 判断を変えたい場合は `decisions.json` を削除するか、テキストエディタで該当エントリを編集してください。
- `unknown_file_action = "keep"` または `"junk"` にすると確認をスキップできます。

## 処理の流れ

```
SendTo で起動
  ↓
対応形式かチェック（非対応 → スキップ）
  ↓
既に無圧縮 ZIP かチェック（output_format=zip かつ該当）
  ├─ ルート直下にファイルあり → スキップ
  └─ ルート直下にフォルダのみ → ルートフォルダ除去のため処理続行
  ↓
7z.exe で一時フォルダに展開
  ↓
junk_dirs に一致するフォルダを削除
  ↓
各ファイルをルールで判定・削除（未分類は確認）
  ↓
空フォルダを削除（remove_empty_dirs=true の場合）
  ↓
ルートに単一フォルダのみ → そのフォルダを新ルートにする
  ↓
無圧縮 ZIP または 無圧縮 RAR を生成（output_format の設定による）
  ↓
元ファイルをゴミ箱へ
  ↓
同じ場所に保存（同名があれば設定に従い回避）
  ↓
タイムスタンプを適用（preserve_timestamp=true の場合）
```

同一パスへ再出力するケースで元ファイルのゴミ箱移動に失敗した場合は、
元ファイルを残したまま変換結果を別名で保存します。

## トラブルシューティング

**「7-Zip が見つかりません」と表示される**
→ 7-Zip を `C:\Program Files\7-Zip\` にインストールするか、`store-zip.exe` と同じ `dist\` フォルダに `7z.exe` を置いてください。

**RAR ファイルが展開できない**
→ RAR5 形式は 7-Zip の最新版が必要です。7-Zip を更新してください。

**日本語ファイル名が文字化けする**
→ 7-Zip を最新版に更新してください。ZIP 形式の旧日本語ファイル名は `-mcp=932` で自動処理しています。

**角括弧 `[` `]` を含むファイル名でエラーになる**
→ ZIP 系ファイルは Python の zipfile モジュールで直接展開するため影響を受けません。
RAR/7z 等は 7-Zip が直接処理します。問題が発生する場合は 7-Zip を最新版に更新してください。

**コンソールの文字が化ける**
→ 起動時に UTF-8 コンソールへ自動切り替えしています。
解消しない場合は `chcp 65001` を実行してから `store-zip.exe` を起動してください。

**一度「削除」にした判断を取り消したい**
→ `dist\decisions.json` をテキストエディタで開き、該当エントリを削除してください。
または `decisions.json` ファイルごと削除すると、次回起動時にすべて再判定されます。

**処理中にスリープ・休止状態になる**
→ 処理中は `SetThreadExecutionState` API でスリープを抑止しています。
それでも問題が発生する場合は、Windows の電源設定を確認してください。

**NAS 上のファイル削除に失敗する**
→ 他のアプリ（7-Zip、ビューアなど）でファイルを開いたままにしていないか確認してください。
SMB のファイルロックが原因の場合、自動リトライを 3 回行います。それでも失敗した場合は元ファイルを残したまま手動削除を案内します。

**「WinRAR が見つかりません」と表示される**
→ `output_format = "rar"` を設定している場合は WinRAR のインストールが必要です。
https://www.rarlab.com/ からインストールしてください。`output_format = "zip"` の場合は不要です。

**「パスワード付きアーカイブは処理できません」と表示される**
→ パスワード保護されたアーカイブは展開できないためスキップされます。パスワードを解除してから再度処理してください。

**1980年以前のタイムスタンプを持つファイルがアーカイブに含まれている**
→ ZIP 形式の仕様上、1980-01-01 以前のタイムスタンプは格納できません。該当ファイルのタイムスタンプは自動的に 1980-01-01 00:00:00 に補正されます（ファイル本体は変更されません）。

## アンインストール

### 1. 「送る」メニューから削除

以下のいずれかの方法で SendTo フォルダを開き、`store-zip.lnk` を削除します。

- エクスプローラーのアドレスバーに以下を貼り付けて Enter:
  ```
  shell:sendto
  ```
- または Win+R → 「ファイル名を指定して実行」に `shell:sendto` と入力

`store-zip.lnk` を削除すると右クリックメニューから消えます。

### 2. store-zip フォルダを削除

`store-zip\` フォルダ（`store-zip.py` や `install.bat` があるフォルダ）をまるごと削除してください。

### 3. インストールされたパッケージの削除（任意）

他のツールで使用していない場合のみ実行してください。

```
python -m pip uninstall pyinstaller send2trash tomli
```

> Python 本体・7-Zip はこのツール専用ではないため、アンインストールは任意です。
